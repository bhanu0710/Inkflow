# Requests for Comments

RFCs document major product capabilities before implementation. They are intended to make scope, tradeoffs, and open questions visible early.

## When to Add an RFC

Add or update an RFC when a feature:

- Introduces a new workflow.
- Changes the event, entity, relationship, or memory model.
- Requires a user-facing product decision.
- Has meaningful security or operational implications.
- Needs review before implementation begins.
